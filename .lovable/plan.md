## สิ่งที่จะเพิ่มใน Pay Dialog (`StaffPayments.tsx` บรรทัด 1018–1117)

### 1. ปุ่ม "โอนเงิน" — แสดงเลขบัญชีพร้อมก้อปปี้
เพิ่มกล่องข้อมูลการโอนด้านบนพื้นที่อัปโหลดสลิป เมื่อเลือก `transfer`:

```text
┌────────────────────────────────────┐
│ ธนาคาร: กสิกรไทย                  │
│ ชื่อบัญชี: นายทูน ใจดี            │
│ เลขบัญชี: 123-4-56789-0   [คัดลอก]│
│ ยอดที่ต้องโอน: 1,250.00 บาท[คัดลอก]│
└────────────────────────────────────┘
```
- ดึงจาก `paySlipDialog.staff_profiles.bank_name / bank_account / staff_name`
- ปุ่มก้อปปี้ใช้ `navigator.clipboard.writeText()` + toast "คัดลอกแล้ว"
- ถ้าไม่มีเลขบัญชี: แสดงข้อความเตือนสีส้ม "ทีมงานยังไม่ได้กรอกเลขบัญชี → ไปที่ทะเบียนทีมงาน"
- ก้อปปี้ "ยอด" จะคัดลอกตัวเลขล้วน เช่น `1250.00` (ไม่มีคอมม่า) เพื่อวางใน app ธนาคารได้ทันที

### 2. ปุ่ม "เงินสด" — ยืนยันแบบมีหลักฐาน
เปลี่ยนปุ่มยืนยันเปล่าๆ เป็นกล่องที่ออกแบบให้ผู้จ่ายมีหลักฐานชัดเจน:
- กล่องสรุป: "จ่ายเงินสด **1,250 บาท** ให้ **นายทูน**"
- ช่อง textarea (optional): "ใครเป็นคนจ่าย / ที่ไหน" เช่น "พี่เอจ่ายที่ออฟฟิศ"
- ช่องอัปโหลด (optional): "ถ่ายรูปลายเซ็นรับเงิน หรือใบรับเงินสด" (ใช้ `SignatureCanvas` หรือกล้องมือถือ) → ถ้าไม่แนบก็ผ่านได้
- ปุ่มเขียวใหญ่: "✓ ยืนยันจ่ายเงินสดแล้ว"
- ถ้าแนบรูป/ลายเซ็น → upload เข้า `documents` bucket, เก็บ path ใน `staff_invoices.payment_slip_url` (ใช้ field เดิม)
- ถ้ามี notes → append ลง `notes` ของ invoice

### 3. ปุ่ม "เครดิต" — ระบุกำหนดชำระ
เครดิตหมายถึง "ยังไม่ได้จ่ายจริง บันทึกเป็นหนี้ก่อน" ออกแบบ:
- กล่องเตือนสีน้ำเงิน: "บันทึกเป็นเครดิต = ยังไม่ตัดเงินสด/บัญชี ระบบจะถือเป็นเจ้าหนี้"
- ช่อง: "กำหนดชำระภายใน" (DatePicker, default = +7 วัน)
- ช่อง textarea: "เหตุผล/หมายเหตุ" เช่น "รอสรุปงานก่อน"
- ปุ่ม: "✓ บันทึกเป็นเครดิต"
- เก็บ `due_date` ลง notes หรือเพิ่ม column ใหม่ (จะใช้ notes ก่อนเพื่อไม่ต้อง migrate)

### Technical Notes
- ไม่ต้องแก้ migration / DB — ใช้ field เดิมทั้งหมด (`payment_slip_url`, `notes`)
- ใช้ `Banknote` / `CreditCard` / `Copy` icon จาก lucide-react (มีอยู่แล้ว)
- เพิ่ม state: `cashNote`, `cashSignatureFile`, `creditDueDate`, `creditNote`
- รีเซ็ต state เมื่อปิด dialog หรือเปลี่ยน payMethod
