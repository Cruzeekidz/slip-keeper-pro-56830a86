

# แผนพัฒนาระบบนำเข้าสลิปจำนวนมาก & ระบบจัดการการเงินครบวงจร

## สถานการณ์ปัจจุบัน

ระบบมีช่องทางนำเข้าข้อมูลอยู่แล้ว:
1. **LINE Bot** — ส่งสลิปทีละใบ AI วิเคราะห์อัตโนมัติ
2. **Bulk Upload** (`/bulk-upload`) — อัพโหลดสลิปรูปภาพ/PDF สูงสุด 20 ไฟล์ต่อครั้ง พร้อม AI วิเคราะห์
3. **CSV Import** — นำเข้าข้อมูลผ่านไฟล์ CSV พร้อม preview และ rollback

## ปัญหาที่ต้องแก้สำหรับการนำเข้าสลิปจำนวนมาก

| ปัญหา | สถานะปัจจุบัน |
|--------|---------------|
| จำกัด 20 ไฟล์ต่อครั้ง | ต้องอัพโหลดหลายรอบ |
| ตรวจซ้ำเฉพาะ transaction_id | สลิปที่ AI อ่าน ID ไม่ได้อาจซ้ำ |
| ไม่รองรับโฟลเดอร์ | ต้องเลือกไฟล์ทีละชุด |
| รายการที่ไม่ชัดเจนปนกับรายการปกติ | ไม่มีระบบ review queue แยก |

---

## Phase 1: ปรับปรุงระบบนำเข้าสลิปจำนวนมาก

### 1.1 เพิ่มจำนวนไฟล์และ Queue System
- เพิ่ม limit จาก 20 เป็น 100 ไฟล์ต่อครั้ง
- ประมวลผลแบบ queue (ทีละ 3 ไฟล์พร้อมกัน) แสดง progress bar
- รองรับการเลือกโฟลเดอร์ (`webkitdirectory`) เพื่ออัพโหลดทั้งโฟลเดอร์

### 1.2 ปรับปรุงระบบตรวจซ้ำ
- ตรวจซ้ำ 3 ระดับ: transaction_id → (amount + date + time) → image hash
- แสดงรายการที่ซ้ำแยกพร้อมตัวเลือก: ข้าม / บันทึกซ้ำ / เปรียบเทียบ

### 1.3 Review Queue สำหรับรายการไม่ชัดเจน
- รายการที่ confidence < 75% เข้า "รอตรวจสอบ" queue อัตโนมัติ
- หน้า Review แสดงสลิปพร้อม dropdown แก้ไขหมวดหมู่ ก่อนยืนยันบันทึก

## Phase 2: พัฒนาสู่ระบบจัดการการเงินครบวงจร

### 2.1 Dashboard การเงินเชิงลึก
- **กำไร-ขาดทุน (P&L)** แยกตาม Entity (ธุรกิจหลัก / BCC Next / คู่ขนาน)
- **Cash Flow Statement** รายเดือน — เงินเข้า vs เงินออก
- **Budget vs Actual** — ตั้งงบประมาณแล้วเปรียบเทียบกับยอดจริง
- **กราฟ Trend** — แนวโน้มรายจ่ายรายเดือน แยกตามหมวดหมู่

### 2.2 ระบบจัดการคู่ค้า (Vendor Management)
- ตาราง `vendors` เก็บข้อมูลผู้ขาย/ผู้รับเงิน (ชื่อ, เลขภาษี, ที่อยู่, บัญชีธนาคาร)
- เชื่อมกับ expenses เพื่อสรุปยอดจ่ายต่อคู่ค้า
- เก็บเอกสารสำคัญ (ใบกำกับภาษี, สำเนาบัตร) ใน Storage

### 2.3 ระบบงบประมาณ (Budget)
- ตาราง `budgets` ตั้งงบรายเดือน/รายไตรมาส แยกตาม entity + category_group
- แจ้งเตือนเมื่อใช้จ่ายเกิน 80% / 100% ของงบ
- แสดงเปรียบเทียบ Budget vs Actual ในหน้า Dashboard

### 2.4 รายงานบัญชี
- **รายงานภาษี**: สรุปยอดรายจ่ายที่หักภาษีได้ แยกตามประเภท
- **รายงานหัก ณ ที่จ่าย (WHT)**: คำนวณอัตโนมัติตามประเภทค่าใช้จ่าย
- **Export** เป็น Excel/PDF สำหรับสำนักบัญชี

### 2.5 เชื่อมต่อ FlowAccount API (อนาคต)
- สร้างรายการค่าใช้จ่ายใน FlowAccount อัตโนมัติ
- ออกเอกสารหัก ณ ที่จ่ายผ่าน API
- Sync สถานะการชำระเงิน

## Phase 3: AI ช่วยวิเคราะห์และตัดสินใจ

### 3.1 AI Financial Insights
- วิเคราะห์ค่าใช้จ่ายผิดปกติ (Anomaly Detection)
- แนะนำการลดต้นทุน จากข้อมูลย้อนหลัง
- พยากรณ์กระแสเงินสด (Cash Flow Forecast)

### 3.2 Auto-Categorization Learning
- เรียนรู้จากการแก้ไขของผู้ใช้เพื่อเพิ่มความแม่นยำ
- สร้าง pattern matching จาก payee_group + merchant

## รายละเอียดทางเทคนิค

### Database Tables ที่ต้องเพิ่ม (Phase 2)
- `vendors` — id, user_id, name, tax_id, address, bank_account, contact
- `budgets` — id, user_id, entity, category_group, month, year, budget_amount
- `financial_reports` — id, user_id, report_type, period, generated_at, file_url

### Edge Function ที่ต้องแก้ไข (Phase 1)
- `analyze-receipt` — เพิ่ม batch mode, ปรับ concurrency

### หน้าใหม่ที่ต้องสร้าง
- `/review-queue` — ตรวจสอบรายการที่ AI ไม่แน่ใจ
- `/vendors` — จัดการคู่ค้า
- `/budget` — ตั้งและติดตามงบประมาณ
- `/financial-reports` — สร้างรายงานบัญชี
